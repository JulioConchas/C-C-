Lab - Networking

Mission: You will network-enable an app, pulling an initial set of data from a web service and storing it in the app's database.

Specification: In this lab, you will modify the app created in the preceding lab. Instead of manually defining your models based on a local array, you will pull a list of names from a web service. You will then save those names to the database.

To successfully complete this lab, you must develop an application that meets the following description:

  • You will use the HTTPClient object to send a GET request to http://bountyhunterapp.appspot.com/bounties which will return an JSON-encoded array of objects.
  • You will modify the code in the root controller. You will replace the code that defines the models based on the local array with code that pulls from the network and processes the array. In each iteration of the loop, create a model and save it.
  • After your loop, you will fetch the collection's data to update the UI.
1. Download the starting code for this lab from S3: 07_networking and unzip the files. Import the project into Titanium Studio. Update the tiapp.xml, if necessary (e.g. to set an appropriate SDK version or build target for your environment).
2. In the root controller, add your code below the comment line. Instantiate a HTTPClient object and point it to the URL noted above. Within its onload handler, parse the returned data. Then create a for-loop to iterate through the results. In each iteration of the loop, create a model instance and save it to the collection. Finally, set the 'seeded' property so this code won't run again. And fetch the collection's data to update the UI.
Hint: You can use portions of the code from the preceding lab example within your loop.
3. Save all the files. Build your app for the iOS simulator or Android emulator. Correct any problems that exist.
Your finished app should match the screen shot above. Names should be displayed. Opening the app again shouldn't add more rows to the table.

Summary

In this lab, you extended the supplied example app and backed its views with data by implementing models and collections.
