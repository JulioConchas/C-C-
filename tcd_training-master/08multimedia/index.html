Background Scaling

  • Background stretched/squeezed to fit
  • On iOS, left/right cap not stretched
  • .. or top/bottom cap
When the background image is larger than the component, it will be squeezed to fit the component's dimensions. Both platforms handle that in essentially the same way, as you can see in the red images in the graphic. But iOS and Android handle background images differently when they're smaller than the component. Look carefully at the following image. Overlaid on top, you see the original small PNG image that is used as the background if the view with the four Xs. In that view, on Android the background image is stretched to fit the width of the component. On iOS, a small portion of the left and right of the image remain un-stretched while the remainder is stretched to fill the width.

Topic

Audio

Sound


// with <Button id="play"></Button>

// in controller:
$.play.addEventListener('click', function() {
  var sound = Titanium.Media.createSound({
    url: 'cricket.wav' // could be http url too!
  });
  sound.play();
});
You can playback local or remote sounds via the createSound() call. The entire sound must be loaded or downloaded before it can be played. This is not a streaming audio player. There is a different control for that purpose.

Recording (iOS)


// with <Button id="record"></Button>

// in controller:
var recorder = Ti.Media.createAudioRecorder({
	compression: Ti.Media.AUDIO_FORMAT_ULAW,
	format: Ti.Media.AUDIO_FILEFORMAT_WAVE
});

$.record.addEventListener('click', function() {
  recorder.start();
  Ti.Media.startMicrophoneMonitor();
});
You can record audio, right now only on iOS. You have options for compression and recording format, among others.

Streaming Audio


// with <AudioPlayer id="player"/> and <Button id='playbtn'/>

// in the controller:
$.player.url = 'http://www.example.com/podcast.mp3';
$.playbtn.addEventListener('click', function() {
  $.player.start();
});

$.win.addEventListener('close', function() {
  $.player.stop(); // stop playback when window closes!
});
You can stream audio from a web URL with the Ti.Media.AudioPlayer object. The AudioPlayer supports pseudo-streaming (of MP3 or other files) as well as HTML Live Streaming.

Video

VideoPlayer


// with <VideoPlayer id="player"/> and <Button id='playbtn'/>

// in the controller:
$.player.url = 'movie.mp4'; // http url would work too!

// set player control style
$.player.movieControlStyle = Titanium.Media.VIDEO_CONTROL_EMBEDDED;
$.playbtn.addEventListener('click', function() {
  $.player.start();
});

$.win.addEventListener('close', function() {
  $.player.stop(); // stop playback when window closes!
})

Streaming support: HTTP Live Streaming (iOS), RTSP (Android), pseudo-streaming

You can show full screen or embedded video players (latter on iOS only) using the VideoPlayer component. You can control the way in which the video playback controls are shown for the player by setting the movieControlStyle property.

Video Events

  • load Video file has finished loading
  • complete Video has finished playing, e.reason on event object will tell you why
  • fullscreen Video player has changed to or from fullscreen mode
e.reason could be something other than Ti.Media.VIDEO_FINISH_REASON_PLAYBACK_ENDED so you should check.

Topic

Capturing Images

From the Gallery


$.galleryButton.addEventListener('click', function() {
  var options = {
    success: function(e) {
      // fired when user selects from gallery
      // e.media == the image data
    },
    cancel: function() {
      // fired when user cancels
    },
    error: function(error) {
      // fired when there's an error
      // error.code is a constant, like Titanium.Media.NO_CAMERA
    },
    allowEditing: true,
    mediaTypes: [Ti.Media.MEDIA_TYPE_PHOTO]
  }
  Ti.Media.openPhotoGallery(options);
});
This opens the platform's native gallery picker. Once the user selects a photo or video, the success callback is fired, inside with e.media is the blob returned.

From the Camera


$.galleryButton.addEventListener('click', function() {
  var options = {
    success: function(e) {
      // fired when user selects from gallery
      // e.media == the image data
    },
    cancel: function() {
      // fired when user cancels
    },
    error: function(error) {
      // fired when there's an error
      // error.code is a constant, like Titanium.Media.NO_CAMERA
    },
    saveToPhotoGallery: true,
    mediaTypes: [Ti.Media.MEDIA_TYPE_PHOTO, Ti.Media.MEDIA_TYPE_VIDEO]
  }
  Ti.Media.showCamera(options);
});
The options supported showCamera and openPhotoGallery are identical. So, you can write a single set of options, then do an if statement to load the gallery or camera. No code duplication that way! The simulator doesn't provide camera support. The Android emulator does, but it gives strange results so don't try to use it.

Ti.Media

  • Media constants
  • Helper properties (e.g. isCameraSupport)
  • Helpful methods (e.g. beep(), vibrate(), etc.)

Demo - Saving images to the gallery

  1. Open gallery in simulator, then close it
  2. Find images using simulator's web browser (e.g. Google Images search)
  3. Press and hold on image
  4. Choose Save image
Demonstrate this, as some students miss the how-to if you just tell them about the procedure.
